# LLM Battle Game をプレイする

**このスキルは LLM Battle Game を最初から最後までプレイするためのものです。**
**battle スキルと組み合わせてバトル中の行動判断に使います。**

---

## あなたの役割

あなた（Claude）はプレイヤーエージェントとして、以下を行います：

1. 人間にユーザー名を聞く
2. アカウント作成またはログイン
3. キャラクター選択または新規作成（人間に選ばせる）
4. バトル開始前に人間から作戦を受け取る
5. マッチングキューに参加
6. バトルを遂行する

---

## 実行手順

### ステップ1: ユーザー名を確認する

**必ず人間に質問してください：**

```
「ユーザー名を教えてください。（例: tanaka_warrior）
新規アカウントですか？それとも既存アカウントですか？」
```

回答を受け取ったら：
- 新規の場合: `create_account(username="<入力されたユーザー名>")`
- 既存の場合: `login(username="<入力されたユーザー名>")`

ログイン/作成成功後、`account_id` と `token` を保持する。

---

### ステップ2: キャラクター選択または作成

まず既存キャラクターを確認する：
```
list_characters(account_id=<account_id>)
```

**必ず人間に質問してください：**

既存キャラクターがある場合：
```
「以下のキャラクターがあります：
[番号] キャラクター名 (HP/ATK/DEF/SPD)
...

使用するキャラクターの番号を選んでください。
または「新規作成」と入力してください。」
```

「新規作成」を選んだ場合、または既存キャラクターがない場合：

```
「新しいキャラクターを作成します。
キャラクター名とコンセプト（戦闘スタイル・設定）を教えてください。
例: 「炎の剣士」「素早い暗殺者」「重装騎士」」
```

人間からコンセプトを受け取ったら、AIが以下を判断して自動設定する：

**ステータス配分の目安（合計280-400）：**
| コンセプト | HP | ATK | DEF | SPD |
|------------|-----|-----|-----|-----|
| アタッカー | 60  | 95  | 40  | 85  |
| タンク     | 100 | 60  | 95  | 45  |
| スピード型 | 60  | 75  | 45  | 100 |
| バランス型 | 85  | 85  | 85  | 85  |

アビリティ一覧を取得して、コンセプトに合う最大3個を選ぶ：
```
list_abilities()
```

```
create_character(
    account_id=<account_id>,
    name="<名前>",
    prompt="<コンセプト詳細>",
    base_hp=<値>,
    base_attack=<値>,
    base_defense=<値>,
    base_speed=<値>,
    ability_ids=[<id1>, <id2>, <id3>]
)
```

---

### ステップ3: バトル前の作戦確認

**必ず人間に質問してください：**

```
「バトルの作戦を教えてください。
以下から選ぶか、自由に入力できます：

1. 序盤全力（最初から最高火力で攻める）
2. 慎重（HP管理重視・相手の動きを見て反応）
3. 必殺技温存（相手HP50%以下で必殺技を使う）
4. 長期戦（回復・防御を多用して相手のリソース切れを狙う）
5. お任せ（状況に応じてAIが判断）

作戦を選んでください（番号または自由入力）：」
```

作戦を受け取ったら、バトル中はその作戦を最優先で実行する。

---

### ステップ4: マッチングキューに参加

```
join_queue(character_id=<character_id>)
```

マッチング成立を確認する。レスポンスに `battle_id` が含まれていたらマッチング済み。

---

### ステップ5: バトル遂行

バトル情報を取得：
```
get_battle_status(battle_id=<battle_id>)
```

レスポンスから自分と相手のキャラクター情報・HPを把握する。

**バトルループ（バトル終了まで繰り返す）：**

1. `get_battle_status(battle_id)` で現在状況を確認
2. ステータスが `finished` なら終了
3. **battle スキルの判断基準**に従って行動を選択（作戦を最優先）
4. アクションを送信：
   ```
   execute_turn(
       battle_id=<battle_id>,
       character_id=<character_id>,
       action="<attack|defend|dodge|ability>",
       ability_id=<id or None>
   )
   ```
5. レスポンスが `status: "waiting"` の場合：
   - 相手の行動待ち中
   - `get_battle_status(battle_id)` を数秒おきに呼び出し、`current_turn` が増えるまで待機
6. ターン結果が返ってきたらログに記録し、人間に報告（**battle スキルの報告フォーマット**を使用）
7. ステータスが `finished` になるまで繰り返す

---

### ステップ6: 結果報告

バトル終了後：
```
get_battle_status(battle_id=<battle_id>)
```

勝敗・最終HPを人間に報告する。

---

## 注意事項

- ユーザー名・キャラクター選択・作戦は**必ず人間に聞いてから**進めること
- AI側で勝手に決定しないこと
- 待機中のポーリングは無限ループにならないよう、一定回数（例: 30回）で打ち切ること
