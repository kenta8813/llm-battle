## 推奨する技術選択

### MCPサーバー: Python FastMCP（実装難易度: 低）
**選定理由**:
- 最もシンプルで直感的な実装
- デコレータ（`@mcp.tool`）だけでLLMツールを定義可能
- 全MCPサーバーの70%がFastMCPを使用
- 公式Python SDKに統合済み

**必要なもの**:
- Python 3.8以上（bashでインストール可能）
- `pip install fastmcp[websockets]`

### データベース: SQLite（実装難易度: 低）
**選定理由**:
- サーバー不要、単一ファイルで管理
- 設定・メンテナンス不要
- ゲームデータには十分な性能
- 600KiB未満の軽量さ

**必要なもの**:
- SQLite（bashでインストール可能）
- Python標準ライブラリにsqlite3が含まれる

### Webサーバー: Express.js + Socket.IO（実装難易度: 中）
**選定理由**:
- リアルタイム双方向通信に最適
- メッセージレイテンシを最大70%削減
- 自動再接続機能
- 豊富なチュートリアル

**必要なもの**:
- Node.js v18以上（bashでインストール可能）
- `npm install express socket.io`

### フロントエンド: React + Vite（実装難易度: 中）
**選定理由**:
- 最新の高速開発環境
- コンポーネントベースで保守性が高い
- WebSocketとの統合が容易

**必要なもの**:
- Node.js（既にインストール済み）
- `npm create vite@latest`
- `npm install socket.io-client`

### マッチング: シンプルなメモリキュー（実装難易度: 低）
**選定理由**:
- ローカル環境では十分
- MMR（レーティング）ベースのマッチング実装可能
- 複雑性を最小化

**必要なもの**:
- 追加ソフトウェア不要（コードのみで実装）
